USING TABLES IN MS ACCESS

    UNDERSTANDING TABLES

  • Access uses tables to define the structure of the data in your database.

  • Each table contains a collection of related information. Access makes it easy to construct tables.

    •

    WHAT IS A TABLE?

  • A table in Access is quite different from the table in real life. Instead of having wooden legs and being used for meals.

  • Access Tables are a grid made up of rows and columns.

  • Tables forms an integral part of database and store all the relevant information.
    •

    ELEMENTS OF A TABLE

  • Any of the other database objects are optional but without a table there can't be a database.

  • They store the complete data in a structured manner. Tables are made up of groups of fields. A field is a specific type of information, such as a student's last name, address or phone number.

    •

    UNDERSTANDING PRIMARY KEY

  • The power of relational database such as MS-Acces comes from its ability to quickly find and bring together information stored in separate tables using queries, forms and reports.

  • In order to do this, each table should include a field or a set of fields that uniquely identify each record stored in the table.

  • This unique field or set of fields is called the primary key of the table.

  • A primary key is a field or combination of fields that uniquely identify each record in a table.
